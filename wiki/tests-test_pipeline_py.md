## Purpose
`tests/test_pipeline.py` tests the compiler pipeline for the Isidore system, focusing on the `plan_pages` function from `isidore.pipeline`. The module verifies that the pipeline correctly selects and processes the most significant modules in a repository, excluding small modules and conceptual nodes. It uses synthetic repository data generated by `_make_repo` to simulate real-world scenarios and validate the pipeline's behavior.

## Architecture
The module consists of helper functions and test cases. The key helper functions are:
- `_node`: Creates a node structure for the synthetic repository.
- `_link`: Creates a link structure between nodes.
- `_make_repo`: Generates a synthetic repository with configurable modules and symbols.
- `_graph`: Loads the graph data from the synthetic repository.
- `_gp`: Returns the path to the graph JSON file.

The test cases validate the `plan_pages` function's behavior, including selecting top modules, handling `top_k` parameters, and recording cross-module dependencies.

## Key entry points
The primary entry point is the `test_plan_pages_selects_top_modules_excluding_small_and_concepts` function, which tests the core functionality of the pipeline. Other test functions validate specific aspects of the pipeline's behavior, such as handling `top_k` parameters and recording dependencies.

## Dependencies
The module depends on:
- `pytest` for testing.
- `isidore.pipeline` for the `plan_pages` function being tested.
- `isidore.render` for rendering functions, though these are not directly used in the tests.

## How to change safely
When modifying `tests/test_pipeline.py`, ensure that:
1. Changes to helper functions (`_node`, `_link`, `_make_repo`, `_graph`, `_gp`) do not break existing test cases.
2. New test cases are added to validate any changes to the pipeline's behavior.
3. The synthetic repository structure remains consistent to avoid test failures due to unexpected changes.
