# PCP_SEAMS — the frozen interface for Proof-Carrying Prose (ADR-0033, phase P0)

This document is the **single contract** the five PCP lanes program against. Everything here is
frozen: schemas, file ownership, function signatures, and the two hard rules (two-oracle honesty and
monotonic escalation). A lane that needs something not defined here must raise it before changing
the seam — do not fork the shape in a lane.

Source of truth for behaviour: `agora/design/ADR-0033-isidore-proof-carrying-prose.md`.
Frozen types + serialization live in `src/isidore/pcp.py`. Golden fixtures live in
`tests/fixtures/pcp/`. The P0 gate is `tests/test_pcp_seams.py` (19 tests, all green).

## Rule 1 — two oracles, declared (fail-closed)

The internal scanner (`graph.py`) emits only `contains` and `imports` edges. Therefore:

- `imports`, `defines`, `exports` → decidable from **graph.json** (nodes + edges). `ORACLE_GRAPH`.
- `calls` → **NOT** in the internal graph. Decide by reparsing the cited file's AST (Python: stdlib
  `ast`, walk the caller symbol's body for a `Call` to the callee). `ORACLE_AST`. A richer external
  graph (Graphify) that DOES carry `calls` edges may be used when present → `ORACLE_GRAPH`.
- `value`, `signature` → AST of the cited file. `ORACLE_AST` (or `ORACLE_LANGSPEC` for non-Python).
- `route` → framework extractor (langspec). `ORACLE_LANGSPEC`.
- `env` → deterministic textual scan. `ORACLE_GREP`.
- `wikichain` (lane D) → resolve against `ctx.pages_state`. `ORACLE_WIKI`.

**A predicate with no verifier for the cited language returns `UNDECIDABLE`, never `TRUE`.** Every
`Verdict` records its `oracle`. UNDECIDABLE degrades a claim to "yellow" (existence-anchored), the
pre-PCP behaviour — it never fabricates confidence.

## Rule 2 — monotonic escalation (I10)

`Mark.disposition` is filled AFTER the LLM call (the generation schema forces the model to address
every mark), but the mark is immutable. `findings = deterministic ∪ LLM`. The model annotates; it
never erases. Clearing a mark is an audited human action (`isidore findings resolve`, lane B/P-INT).

## Typed-claim grammar (lane A owns the parser; format frozen here)

Today's claims block is `<statement> | <path:line>`. PCP adds an optional **third** field:

```
<statement> | <path:line> | <kind>:<arg1>;<arg2>;...
```

- Third field absent/empty → existence-anchored (yellow), fully backwards compatible.
- `;` separates predicate args (NOT `,` — commas break TOON). Kinds and arg semantics: see
  `pcp.PREDICATE_KINDS` and the `Predicate` docstring. Parse with `pcp.parse_predicate`, serialize
  with `Predicate.serialize` (round-trips — test-guarded).

Example (from the golden page `tests/fixtures/pcp/repo/wiki/svc.md`):
```
authenticate verifies the JWT by calling verify_jwt | svc/auth.py:14 | calls:authenticate;verify_jwt
```

## Certificate (`<page>.md` → `<page>.md.cert.json`, alongside the page)

**JSON, not TOON** — a certificate is machine-read by `isidore verify`; JSON re-parses with the
stdlib. TOON stays the LLM-facing format. Shape = `pcp.Certificate` (see the golden
`svc.md.cert.json`, generated by `pcp.write_certificate`). Fields: `page`, `version` (`pcp-1`),
`graph_commit`, `prose_sha256` (tamper anchor — full sha256 of the parsed prose, `pcp.prose_hash`),
`claims` (list of `ClaimVerdict`), `marks`, `violations`, `mass` (green/yellow/gray), and
`child_cert_hashes` (lane D composed integrity). Persistence: `pcp.write_certificate` /
`read_certificate` (round-trips byte-deterministically — sorted keys).

## Contracts (`contracts.json` in the wiki dir)

Shape = `{"contracts": [pcp.Contract, ...]}`. `pcp.read_contracts` / `write_contracts`. A `Contract`
is a promoted typed claim: `{id, predicate, evidence, promoted_by, promoted_at (ISO, caller-supplied),
note}`. The gate `contracts.verify_contracts(contracts, ctx)` returns `[(Contract, Verdict)]`; any
`FALSE` fails `isidore verify --contracts` (exit nonzero). No wall-clock in the seam — the CLI passes
timestamps in.

## Marks (lane C output; also the golden `marks.json`)

Shape = `pcp.Mark`: `{family (entropy|sink|topology), file, line, reason, severity (info|warn|danger),
disposition}`. `scan(root, ctx) -> list[Mark]`. Marks enter the generation call as **DATA** (the
existing FACTS channel, invariant I8), never as instructions. `disposition` starts empty; see Rule 2.

## wiki:// chains (lane D)

A pyramid claim (level ≥ 2) cites `wiki://<page>#<claim-id>` in its `evidence`. `pcp.parse_wiki_uri`
splits it. Lane D registers a verifier under `pcp.WIKI_VERIFIER_KIND` (`"wikichain"`, a reserved kind
— not LLM-facing) that resolves against `ctx.pages_state`: the cited claim must exist, be non-stale,
and carry verdict TRUE. Certs compose via `child_cert_hashes`; staleness propagates upward with 0 LLM.

## Pipeline hooks (lane A wires; signatures frozen)

Two call-sites in `pipeline.compile_wiki`'s per-page loop (see `pipeline.py:628-687`):

1. **pre-generation** — compute marks for the page's files and add them to the prompt as FACTS:
   `marks = detectors.scan(repo, ctx)` (lane C). Filtered to the page's files by lane A.
2. **post-parse** — after `parse_claims_block` + `parse_findings_block`, before writing the page:
   - lane A: verify typed claims → `ClaimVerdict`s; classify prose mass; `build_certificate(...)`.
   - lane B: `violations = reconcile(prose, findings, claims, marks)`; escalate per Rule 2.

Lane A implements hook #1's wiring and its own verify pass in P0-frozen call shapes; **P-INT** connects
lanes B and C into these sites (mechanical — the signatures already match). `CompileResult` gains
fields in lane A: `certificates: list[str]`, `contracts_broken: list[str]`, `marks_raised: int`,
`reconcile_violations: int`, `verified_mass: dict` (declare in the dataclass; keep defaults so dry-run
and existing tests are untouched).

## CLI

`cli.main` runs a registrar loop (written once in P0): `for _mod in (verify, contracts, pyramid,
humanpack): _mod.register_cli(sub)`. Each lane owns its subcommand inside its module. Lane C has no
subcommand (its marks flow through the pipeline; a debug `scan-marks` is optional and, if added, goes
in detectors.py with its own `register_cli`, appended to the loop by that lane — the one allowed
cli.py touch, a single line).

## How each lane starts (all depend ONLY on P0 = T-1dc9)

- **A**: implement the eight verifiers in verify.py against `tests/fixtures/pcp/` (the golden cert is
  the expected output); extend claims.py's parser; add the CompileResult fields; wire hook #1 + its
  verify pass in pipeline.py. Gate includes the F0 novelty pass (Chinese sources).
- **B**: implement `reconcile()` + contracts against fixtures and `golden_vigil/`; never import
  pipeline/claims/verify (test-guard it).
- **C**: implement `scan()` (three families) + the langspec sinks table; measure FP on ≥2 clean repos.
- **D**: implement `plan_pyramid()` + the `wikichain` verifier against fixtures + `pyramid_config.json`.
- **E**: implement `render_pack()` against the golden artifacts (page + cert + graph + contracts);
  0-LLM test-guarded; deterministic output.
- **P-INT** (T-aad2): wire B and C into the pipeline hooks; gates, CI recipes, dogfood ≥3 repos,
  measurements table into ADR-0033.

Golden fixtures recap (`tests/fixtures/pcp/`): `graph.json`, `repo/svc/{auth,tokens}.py` (line
numbers load-bearing), `repo/wiki/svc.md` (typed-claims page), `repo/wiki/svc.md.cert.json` (real
hashes), `contracts.json`, `marks.json`, `pyramid_config.json`.
